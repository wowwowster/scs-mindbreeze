1 - Architecture
eZ Publish Connector is part of SCS solution on external server adapting the GSA to Drive folders. It indexes eZ Publish folders or files with specific metadata and pushes indexed content to the Google Search Appliance.
2 - Additional requirements
In addition to the SCS global requirements listed in the SCS documentation, the following additional requirements apply for this connector:
2.1 - Hardware
none
2.2 - Software
none
2.3 - Network
Google servers must be reachable from the connector machine.
3 - Connector SCS functionalities
The eZ Publish connector can index eZPublish documents.
4 - Connector configuration
4.1 - Connector definition
Connector ID: Connector instance name which will be used as reference. This field is mandatory and will be locked once the connector instanciate.
Usage: Defines what this connector will be used for. Only one connector can be configured for authentication. There can be as many group retrievers as necessary. This field is mandatory.
Namespace: Namespace to use for user/group names. Namespaces allow disambiguation for ACEs with the same name but originating form different systems e.g: many systems contain a group named "Administrators" which is likely to contain different users.
4.2 - eZ Publish parameters
database type: The database type your EzPublish is built with. This field is mandatory
Connection String: JDBC connection string: <server>:<port>/<db>?<options>. This field is mandatory.
User: A SQL user with read permission on all required tables. This field is mandatory.
Password: Above user's password. This field is mandatory.
Hostname: Fully qualified EzPublish hostname e.g.: http://<yourezpublish>.com. This field is mandatory.
Site language: Will not get attributes with another language defined.
Only index searchable attributes: Only get attributes that can be found with navigation.
Only index published content: Only get published documents.
Url separator: Separator used in URL to replace illegal characters, default \'-\'.
Root Node: If EzPublish contains more than one site, you can define a root node.
Only referenced files: Only get files if they are linked from an index document. Default false.
get full metadata: Will allow recursively looking for metadata about objects linked to an object.
Direct index types: List of data types relevant for indexing (article, newspaper, forum,...). This field is mandatory.
Binary types: Non-textual data types. If a type is on this list but not on the index types list, its elements will only be indexed when other types are referring to them.
4.3 - Indexing
Feed files conservation period (days): Defines how long feeds will be stored on disk. 0<=> immediate delete. Defaults to 0.
Max feed size (MB): Maximum size for a feed file. GSA does not support files larger than 1GB. Defaults to 100MB
Max document size (MB): Maximum size for an individual document content. Defaults to 100MB.
Feed dates format: Format for date metatags. Defaults yyyyMMdd.
Excluded MIME types: List of MIME types that should'nt be indexed.
Constant meta tags: List of metadata tags that will be added to all documents (observing format: name=value ; name cannot contain an equal sign).
Meta tags renaming: List of meta tags which name should be changed. For instance if you want the meta tag named Creator to be change to Autor, enter Creator= Author.
Socket timeout (seconds): Socket timeout configured for HTTP client. Default to 1 minute.
Max indexing threads: Size of the thread pool dedicated to indexing. Defaults to 2.
Auth method: Authentication method. Default to httpsso
Super-user group: If a group has access to all documents, specify this parameter and the group will be added to all documents ACLs.
User ACEs as Group ACEs: When indexing multiple repositories, if a user has different names accross repositories, indexing user ACEs as group ACEs is necessary because the GSA only allows one verified identity per credential group.
JAVA Options: Misc options that are passed JAVA at runtime(-D options, -XX options,...)
IDE time: When traversal completes, this setting determines how long the connector will wait before it starts traversing the repository again (expressed in hours).
Maximum throughput: Determines the traversal rate (in kB/S). Values lower than, or equal to zero will set an infinite traversal rate.
Do not index: Prevents the indexer from connecting to the source system during certain periods of time.
